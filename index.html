tsoa

Build swagger-compliant REST APIs using TypeScript and Node

View on GitHub Download .zip Download .tar.gz

What is it?

  • TypeScript controllers (classes) and models (interfaces) as a single source of truth for your RESTful API
  • Swagger spec generation: enables documentation and client SDK generation
  • Route generation: middleware routes generated from controllers

Philosophy

  • Rely on TypeScript type annotations to generate API metadata if possible
  • If regular type annotations aren't an appropriate way to express metadata, use decorators
  • Use jsdoc for pure text metadata (e.g. endpoint descriptions)
  • Minimize boilerplate
